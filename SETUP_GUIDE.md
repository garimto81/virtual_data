# 🎯 독립 실행형 카메라 업데이트 도구 설정 가이드

## 📋 개요
기존 Index 시트에 카메라 정보를 일괄 업데이트하는 1회성 독립 Apps Script입니다.

## 🚀 설정 단계

### 1. 새 Google Apps Script 프로젝트 생성
1. [https://script.google.com](https://script.google.com) 접속
2. "새 프로젝트" 클릭
3. 프로젝트 이름을 "카메라정보업데이트" 등으로 설정

### 2. 코드 복사 및 설정
1. `standalone_camera_updater.gs` 파일의 전체 내용을 복사
2. Apps Script 에디터에 붙여넣기
3. **중요**: `SPREADSHEET_ID` 값을 실제 구글 시트 ID로 변경

```javascript
// 이 부분을 수정하세요
const SPREADSHEET_ID = 'YOUR_ACTUAL_SPREADSHEET_ID_HERE';
```

### 3. 권한 설정
1. 저장 (Ctrl+S)
2. "권한 검토" 버튼 클릭
3. Google 계정으로 로그인
4. "고급" → "안전하지 않은 페이지로 이동" 클릭
5. "허용" 버튼 클릭

### 4. 웹 앱으로 배포
1. "배포" → "새 배포" 클릭
2. 유형: "웹 앱" 선택
3. 설명: "카메라 정보 업데이트" 입력
4. 액세스 권한: "모든 사용자" 선택
5. "배포" 클릭
6. **중요**: 생성된 웹 앱 URL 복사

### 5. HTML 파일 수정
1. `update_camera_info.html` 파일 열기
2. 129번째 줄의 Apps Script URL을 새로 생성한 URL로 변경

```html
<input type="text" id="scriptUrl" value="새로_생성한_웹앱_URL_여기에" style="width: 600px;">
```

## 🔧 테스트 방법

### Apps Script 에디터에서 테스트
1. `testConnection` 함수 선택
2. "실행" 버튼 클릭
3. 로그에서 연결 상태 확인

### 웹 인터페이스에서 테스트
1. `update_camera_info.html` 파일을 브라우저에서 열기
2. "데이터 확인" 버튼 클릭
3. 시트 연결 및 데이터 로딩 확인

## ⚡ 실행 방법

### 방법 1: 웹 인터페이스 사용 (권장)
1. `update_camera_info.html` 파일 열기
2. "데이터 확인" → "미리보기" → "실행" 순서로 진행

### 방법 2: Apps Script에서 직접 실행
1. Apps Script 에디터에서 `manualUpdate` 함수 선택
2. "실행" 버튼 클릭
3. 로그에서 진행상황 확인

## 📊 기능 설명

### 주요 기능
- **자동 번호 할당**: 0001, 0002, 0003... 형식으로 순차 할당
- **카메라 이름 자동 가져오기**: Type 시트 A2, A3에서 자동 읽기
- **배치 업데이트**: 대량 데이터를 효율적으로 처리
- **에러 처리**: 실패한 행은 건너뛰고 계속 진행
- **업데이트 로그**: UpdateLog 시트에 작업 기록 자동 저장

### 업데이트 대상
- **J열**: "Cam1+Cam2" 형식
- **K열**: 첫 번째 카메라 이름
- **L열**: 첫 번째 카메라 번호 (0001)
- **M열**: 두 번째 카메라 이름
- **N열**: 두 번째 카메라 번호 (0002)

## ⚠️ 주의사항

1. **백업**: 실행 전 Google Sheets 백업 권장
2. **1회성**: 이미 카메라 정보가 있는 행은 건너뜀
3. **권한**: Google Sheets 편집 권한 필요
4. **시트 구조**: Index, Type 시트가 존재해야 함

## 🐛 문제 해결

### "권한 없음" 오류
- Apps Script 배포 시 "모든 사용자" 권한 설정 확인
- 웹 앱 URL이 올바른지 확인

### "시트를 찾을 수 없음" 오류
- SPREADSHEET_ID가 올바른지 확인
- Index, Type 시트 이름이 정확한지 확인

### "업데이트할 행 없음"
- Index 시트에 이미 모든 카메라 정보가 입력되어 있음
- handNumber(A열)는 있지만 cam1no(L열)가 비어있는 행이 있는지 확인

## 📈 성능 최적화

- **배치 처리**: 한 번에 여러 행 업데이트
- **에러 처리**: 일부 실패해도 전체 작업 계속
- **로그 기록**: 작업 결과 자동 기록

## 🔍 디버깅

- 연결 상태
- 처리된 행 수
- 오류 메시지
- 실행 시간
